30 years, one full circle

timeline
    title From server HTML, to the client, and back
    1996 : Classic ASP : Server renders HTML
    2002 : ASP.NET Web Forms : Postbacks & ViewState
    2006 : jQuery / Underscore / CoffeeScript : Sprinkles of JS
    2013 : Angular / React : The SPA era — logic moves to the client
    2020 : htmx / Alpine AJAX : HTML over the wire, again
ASP and Web Forms: server rendered everything. Then jQuery let us sprinkle behaviour. Then SPAs swallowed the world — and we moved all the logic to the browser. Alpine AJAX is the pendulum swinging back: keep the snappy UX, put the logic back where it belongs.

HTML fragments

  B[Browser] -- request --> S[Server]
  S -- HTML fragment --> B

Logic: server ✅ · UX: great ✅

SSR: logic on the server, but full reloads. SPA: gorgeous UX, but now you maintain logic, validation and routing TWICE — and a JSON API in between. Alpine AJAX: the server still renders HTML; Alpine swaps in just the bits that changed. One codebase, one source of truth.

The pitch

The server is the best place for your logic.

Send HTML, not JSON.

Keep the SPA feel — with attributes, not a framework.

No build step. No client state to sync. No second codebase. Your ModelState validation, your auth, your domain rules — all stay on the server. Alpine AJAX is ~5 attributes. That's the whole API.
